# 贡献指南

感谢你参与 `douyin-mcp`。提交代码、文档或其他可受版权保护的内容前，请阅读本指南和 [PLATFORM_COMPLIANCE.md](PLATFORM_COMPLIANCE.md)。

## 贡献许可

本项目采用 `AGPL-3.0-only`。除非另有书面协议，你提交并由项目接收的贡献也将按 `AGPL-3.0-only` 提供，即“贡献许可与项目对外许可一致”。

提交贡献即表示你确认：

1. 你本人创作了该贡献，或者已经获得按 `AGPL-3.0-only` 提交该贡献所需的全部权利；
2. 该贡献不会故意混入许可证不兼容的代码、文档、素材或数据；
3. 如果内容来自第三方、生成式 AI 或其他工具，你已经完成必要的来源、版权和许可证审查，并会保留所需声明；
4. 你理解贡献一旦按 AGPL-3.0-only 发布，已获得的许可证通常不能对既有接收者撤回。

如你代表雇主、客户、学校或其他组织贡献，请先确认你有权这样做。

## 平台合规要求

提交涉及抖音访问或数据处理的变更前，请特别核对《“抖音”用户服务协议》第 2.4、5.1、5.3、7.1 条及创作者中心适用的专项规则。AGPL 允许商业使用项目代码，但不代表平台允许自动化访问、平台外处理或展示、向第三方提供信息或商业使用平台数据。

贡献不得增加或帮助实现以下能力：

- 绕过登录、扫码、安全验证、验证码、权限、风控或技术保护；
- 隐藏自动化特征、规避平台检测或模拟未经授权的客户端；
- 未经授权访问他人账号、批量托管账号或收集他人数据；
- 调用未公开私有接口、利用漏洞或破坏平台正常运行；
- 将项目包装成抖音官方、授权或合作产品。

涉及浏览器访问、页面解析、数据字段或同步策略的变更，应在说明中写明数据来源、缺失语义、请求频率、账号与隐私风险、MCP 客户端或模型服务可能接收的数据，以及如何避免扩大现有能力边界。

## 敏感信息

请勿提交 Cookie、Token、浏览器 profile、扫码登录记录、真实账号数据、数据库、导出文件、HAR、Trace、截图或其他可识别个人或账号的信息。测试样例必须去标识化并确认有权公开。

## 提交前检查

```powershell
python -m compileall -q src
python -m pytest -q
```

同时确认新增依赖具有明确、兼容的许可证，并在引入第三方代码或资源时保留其版权、许可证和 NOTICE 要求。
